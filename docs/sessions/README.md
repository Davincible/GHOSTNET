# Session Logs

This directory contains logs of significant development/architecture sessions.

## Purpose

Session logs capture:
- Decisions made and their rationale
- Assumptions that need verification
- Risks identified
- Open questions
- Artifacts created

## Template

When creating a new session log, include:

```markdown
# Session Log: [Title]

**Date:** YYYY-MM-DD  
**Type:** [Architecture/Implementation/Debug/Review]  
**Status:** [In Progress/Completed]

## Session Overview
Brief description of what was accomplished.

## Key Decisions Made
Document significant decisions with rationale.

## Assumptions Made
Track assumptions that need verification.

## Open Questions
Questions that need answers.

## Artifacts Created
Files/documents produced.

## Next Steps
What comes next.
```
